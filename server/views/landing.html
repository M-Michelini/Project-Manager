  • I can POST a username, email, password and timezone to api/register to hash the password, then store that info in a mongo database.
  • I can POST a username and password to api/login to hash the password, then compare it to the one in the a mongo database.
  • If the POST to api/login is successful, the server will respond with the username, email, timezone and a jsonwebtoken to be stored in the clients cookies. This allows validation without a password.
